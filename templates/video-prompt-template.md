# AI Construction Video — Video Prompt Template

---

# Core Principle

Video generation should animate the transformation between Start Frame and End Frame.

The video model should not invent the construction logic.

The construction logic must already be defined by the scene breakdown and image sequence.

---

---

# Global Rules

Every video prompt must include:

- Start state
- End state
- One consistent worker
- Clear construction action
- Fast realistic timelapse
- Strong ASMR work sounds
- Static camera unless intentionally specified
- No cuts
- No transitions
- No teleporting materials

---

# Worker Consistency

Default worker:

- Grey hoodie
- Black baseball cap
- Black work pants

Rules:

Use one worker only.

Do not introduce multiple workers.

Do not change outfit.

Do not change body type.

---

# Camera Rules

Default:

Cinematic static shot.

Requirements:

- No camera movement
- No zoom
- No transitions
- No camera shake
- Same perspective as reference images

Exceptions:

- POV entrance scene
- Final walkthrough
- Reveal scene

---

# Timelapse Rules

The scene should play as:

Fast realistic timelapse.

Use phrases:

- Fast realistic timelapse
- Accelerated construction workflow
- Visible full process
- Rapid but physically believable motion

---

# ASMR Rules

Include realistic work sounds.

Examples:

- Shovel digging
- Hammer impacts
- Drilling
- Welding
- Footsteps
- Gravel movement
- Wood placement
- Metal contact
- Material handling

Use phrases:

- Strong ASMR work sounds
- Realistic tool impact sounds
- Natural construction audio

---

Strong ASMR emphasis on [RELEVANT_WORK_SOUNDS].

No cuts.

No transitions.

No camera movement.

No zoom.

No teleporting materials.

No sudden completion.

# Quality Checklist

Before accepting generated video:

- Does it follow the Start Frame?
- Does it reach the End Frame?
- Is the worker consistent?
- Is the action physically believable?
- Is the timelapse visible?
- Are materials moved through real action?
- Is the camera stable?
- Are there no cuts or transitions?
- Is the scene visually progressing?

---

# Failure Recovery

If output fails:

## Problem: Too slow

Fix:

Add stronger timelapse language.

## Problem: Materials teleport

Fix:

Specify step-by-step physical action.

## Problem: Camera moves

Fix:

Repeat static camera rules.

## Problem: Worker changes

Fix:

Emphasize one worker only.

## Problem: End frame not reached

Fix:

Clarify final state and construction objective.

---

# Common Mistakes

Avoid:

- “Build a house” vague prompts
- Multiple workers
- Missing Start Frame / End Frame
- Weak timelapse
- No ASMR instruction
- Camera movement
- Sudden completion
- Too many actions in one scene

# Example

Concept:

Hidden Waterfall House

Scene:

Excavate entrance behind waterfall

Start state:

Natural rock wall behind waterfall with no entrance.

End state:

A rough tunnel entrance carved into the rock wall with debris around the opening.

Prompt:

Cinematic static shot of a waterfall cliffside.

A single worker wearing a grey hoodie, black baseball cap, and black work pants uses a pickaxe and shovel to carve a tunnel entrance into the rock wall behind the waterfall.

The scene plays as a fast realistic timelapse.

The worker repeatedly breaks rock, removes debris, and gradually forms the tunnel entrance.

Strong ASMR emphasis on pickaxe impacts, stone cracking, shovel scraping, and falling debris.

No cuts.

No transitions.

No camera movement.

No zoom.

No teleporting materials.

The final frame should match the tunnel entrance end frame.

---

# Notes

The video prompt controls motion.

The image prompt controls structure.

Never rely on video generation to invent the full scene.

Always guide it with Start Frame and End Frame.
